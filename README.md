The project generates images from the training dataset with input as a piece of text describing that image.

Training is collected from 102 different species of flowers and the corresponding description of each image.

Link datasets = https://drive.google.com/drive/folders/173rCnPRMOd4kcYDxc2saaRfl7kBVa6TF

Text to Image introduction:
The goal is to create automatic images from the text description. In Text-to-Image Synthesis, an AI system that receives input is a paragraph describing the text and creating a corresponding image based on the content of that description.
Applications of Text-to-Image Synthesis may include:
- Create images for game models or virtual environments from the description of the user.
- Create illustrations for the story or article.
- Support the creation of diverse data for machine learning data kit.

1. Overview of GAN:

GAN takes a noise vector as input, which is transformed into input data when passed through the Generator to the Discriminator model.

The Generator attempts to learn information through the loss function: L_G (z)=-log⁡(D(G(z))) to generate realistic data that deceives the Discriminator. 
Meanwhile, the Discriminator learns based on data generated by the Generator and real data from the input, improving through the loss function:
L_D (z,x)=-ylog⁡(D(x))-(1-y)log⁡(1-D(G(z))) to prevent being misled by the Generator and provide accurate predictions about the labels of the data passed through it.

The noise vector is converted into a tensor and passed through a deep convolutional network, resulting in a tensor corresponding to the input size of the image.

- Discriminator:

![image](https://github.com/NguyenTienAnh8365/Text-to-Image-Synthesis-using-DCGAN/assets/153089067/93592ec0-177e-403e-8d5e-ee4973d3cc61)


The Discriminator receives images from two datasets: fake images generated by the Generator and real images from the dataset. These images are passed through a deep convolutional network, and the output is transformed into real/fake labels.
